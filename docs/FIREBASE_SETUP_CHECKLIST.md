# Firebase Console Setup Checklist

Complete these steps in order before running the app in production.

## 1. Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/).
2. Click **Add project** â†’ name it (e.g. `ortho-clinic-prod`).
3. Disable Google Analytics if not needed (optional for HIPAA-style workflows).
4. Wait for project provisioning.

## 2. Register Apps (Web + Windows)

### Web

1. Project Overview â†’ **Add app** â†’ **Web** (`</>`).
2. Register nickname: `ortho-clinic-web`.
3. **Skip** the Console steps â€śAdd Firebase SDKâ€ť (`npm install firebase` or `<script>` tags). Those are for plain JavaScript websites, not Flutter.
4. Paste the `firebaseConfig` values into `lib/firebase_options.dart` (already done for `ortho-clinic-prod`), or run `flutterfire configure` to regenerate that file.

### Windows (optional native Firebase)

1. For Flutter Windows desktop, Firebase uses the same Web config via `firebase_options.dart` generated by FlutterFire.
2. Ensure **Windows** is enabled in your Flutter project (`flutter create --platforms=windows .`).

## 3. Authentication

1. **Build â†’ Authentication â†’ Get started**.
2. Enable **Email/Password** sign-in provider.
3. Create two staff accounts manually (or via Admin SDK):
   - `reception@clinic.local` â†’ assign role in Firestore (step 5).
   - `doctor@clinic.local` â†’ assign role in Firestore.
4. Under **Settings â†’ Authorized domains**, add your production web domain when deploying.

## 4. Cloud Firestore

1. **Build â†’ Firestore Database â†’ Create database**.
2. Choose **Production mode** (security rules will be deployed from `firebase/firestore.rules`).
3. Select a region close to your clinic (e.g. `europe-west1`).
4. Deploy rules and indexes:

   ```bash
   npm install -g firebase-tools
   firebase login
   firebase init firestore   # select existing project, use firebase/ folder
   firebase deploy --only firestore:rules,firestore:indexes
   ```

5. **Offline persistence**: enabled in app code (`FirebaseInitService`). No console toggle required.

## 5. Seed User Documents (RBAC)

For each Auth user, create a document in `users` collection:

| Document ID | Fields |
|-------------|--------|
| `{auth_uid}` | `uid`, `name`, `email`, `role`: `receptionist` or `doctor` |

Example (Firestore Console â†’ Add document):

```json
{
  "uid": "abc123",
  "name": "Dr. Ahmed",
  "email": "doctor@clinic.local",
  "role": "doctor"
}
```

**Critical:** `role` must match exactly `receptionist` or `doctor` â€” security rules read this field.

## 6. Cloud Storage

1. **Build â†’ Storage â†’ Get started**.
2. Start in **production mode**.
3. Deploy `firebase/storage.rules`:

   ```bash
   firebase deploy --only storage
   ```

4. Default bucket: `gs://YOUR_PROJECT_ID.appspot.com` â€” used for X-ray paths under `xrays/{patientId}/{visitId}/`.

## 7. FlutterFire Configuration

From project root:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

Select your Firebase project and platforms **Web** + **Windows**. This generates `lib/firebase_options.dart`.

## 8. Composite Indexes

If queries fail at runtime, Firestore logs a link to create indexes. Pre-deploy `firebase/firestore.indexes.json` (included in repo).

Required indexes:

- `live_queue`: `status` ASC + `created_at` ASC
- `invoices`: `date` ASC (range queries)
- `expenses`: `date` ASC
- `visits` (collection group): `date` DESC

## 9. Billing & Quotas

1. Upgrade to **Blaze (pay-as-you-go)** only if you exceed Spark free tier limits.
2. Set **Budget alerts** in Google Cloud Console for Storage (5 GB free) and Firestore reads.
3. Image compression is enforced client-side before upload (see `ImageCompressService`).

## 10. Production Hardening (Recommended)

- [ ] Enable **App Check** for Web/Windows.
- [ ] Ensure financial dashboard access is restricted to doctor users and enforce this in authentication/authorization flows.
- [ ] Configure **Firebase Hosting** for web deploy: `firebase init hosting`.
- [ ] Enable daily **Firestore backups** (Blaze plan).
- [ ] Review Storage CORS if loading images from custom domains.

## 11. Run the App

```bash
flutter pub get
flutter run -d chrome
flutter run -d windows
```
